Task graphs for humans and AI agents

Coordinate work across humans and AI. Dependencies respected. No double-work. Just flow.

cargo install --git https://github.com/graphwork/workgraph 
$ wg add "Design the API"

Newly unblocked:
  build-backend

Why workgraph?

Dependency Graphs

Tasks block other tasks. When you complete work, downstream tasks automatically unblock. No manual tracking needed.

Multi-Agent Coordination

Humans and AI agents work together. Atomic claims prevent double-work. Heartbeats detect dead agents.

Project Analysis

Find bottlenecks, trace critical paths, forecast completion. Understand your project's health at a glance.

AI-Native Design

Context flow, trajectories, and skills. AI agents know what to work on, what context is available, and when to stop.

Plain Text Storage

Everything in .workgraph/graph.jsonl. Human-readable, git-friendly, easy to hack on.

Simple Protocol

wg ready → wg claim → wg done. That's it. Works in any shell, any workflow.

How It Works

Tasks flow through the graph. Complete one, and blocked tasks become ready.

Get Started

1

Install

cargo install --git https://github.com/graphwork/workgraph
2

Initialize

$ cd your-project
$ wg init
Created .workgraph/
3

Add tasks

$ wg add "Set up CI pipeline"
$ wg add "Deploy to staging" --blocked-by set-up-ci-pipeline
$ wg add "Run integration tests" --blocked-by deploy-to-staging
4

Work

$ wg ready                    # what can I do?
$ wg claim set-up-ci-pipeline --actor erik
# ... do the work ...
$ wg done set-up-ci-pipeline  # unblocks downstream

Human + AI Coordination

Workgraph shines when humans and AI agents work together on the same project.

Human Plans, Agent Executes

# You define the work
$ wg add "Refactor auth module" --skill rust
$ wg add "Update tests" --blocked-by refactor-auth-module
$ wg add "Update docs" --blocked-by refactor-auth-module

# Agent runs through ready tasks
$ wg agent --actor claude --max-tasks 10

Parallel Agents

# Multiple agents, no conflicts
$ wg coordinate --max-parallel 3
Parallel execution slots:
  1. implement-api (rust)
  2. write-docs (documentation)
  3. setup-ci (devops)

# Each agent claims different work
$ wg agent --actor agent-1 &
$ wg agent --actor agent-2 &
$ wg agent --actor agent-3 &

Mixed Workflow

# Human takes design decisions
$ wg claim design-api --actor erik
# ... human does design work ...
$ wg done design-api

# Agent implements once design is done
$ wg agent --actor claude
-> Working on: implement-api
   Executing: cargo build
   Completed: implement-api

Works with AI Coding Assistants

Workgraph includes a skill that teaches AI assistants the full protocol. Drop it into your project and any AI can participate.

Claude Code

$ mkdir -p .claude/skills
$ cp workgraph/.claude/skills/wg.md .claude/skills/

Other Agents (OpenCode, Codex, etc.)

$ cat workgraph/.claude/skills/wg.md >> AGENTS.md

The skill covers: claim/work/done protocol, progress logging, planning with dependencies, analysis commands, and multi-agent coordination rules.
